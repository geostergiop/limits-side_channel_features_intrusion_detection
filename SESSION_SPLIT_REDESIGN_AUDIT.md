# Session Split Redesign Audit

## Status

The session experiment implementation now uses protocol-specific frozen manifests.
All session results generated by the previous repeated grouped-holdout protocol are
invalid for comparison with this redesign and must be rerun.

## Implemented Protocols

- `capture_disjoint_5fold`: primary unseen-capture protocol. Each malware capture
  is held out exactly once; every benign capture is tested; train, validation, and
  test captures are pairwise disjoint.
- `within_capture_temporal`: secondary seen-capture upper bound. Sessions are
  ordered by first packet timestamp and `session_id`, then partitioned 60/20/20
  independently inside every capture. A session is never split.
- `balanced`: capture assignment occurs first, followed by deterministic equal-
  class validation/test quotas. The same frozen IDs are used across compatible
  feature sets and algorithms.
- `deployment`: complete held-out-capture samples preserve corpus prevalence.
  Thresholds maximize validation recall subject to the configured validation FPR
  ceiling; test data are evaluated only after threshold selection. PR-AUC and
  ROC-AUC are computed from untouched test scores.

Consequences:

- Whole-session `session_sequence` and `behavior_window` selection both use the
  six-packet threshold. This deliberately supersedes the eight-packet threshold in
  the original redesign handoff and retains 11,247 eligible Hancitor sessions.
- Eligibility is part of both the manifest filename payload and cohort hash.
  Existing eight-packet manifests therefore cannot be reused as six-packet results.

## Reporting and Artifact Integrity

- Protocol-qualified result, checkpoint, report, fine-tune, and manifest paths
  prevent old/new or capture/temporal rows from being loaded together.
- Existing-result loading rejects every row whose `split_mode` differs or is absent.
- Manifest creation is lock-serialized; writes use temporary files and atomic
  replacement.
- Reports use fold median, IQR, minimum, maximum, and explicitly labelled pooled
  secondary metrics. Standard deviation remains in JSON as heterogeneity, never as
  an expected score range.
- Result rows disclose held-out family, full test support/prevalence, budgeted LLM
  support/prevalence, threshold metadata, ranking metrics, runtime, and paired
  fold/sample differences.
