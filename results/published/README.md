# Published Result Summaries

This directory contains six compact exports of the primary capture-disjoint
Phase 7 session experiments, one provenance-linked Phase 4E summary, one
author-supplied supplemental GPT-5.4 bin-width sweep, and one audited input-
requirement summary.
Raw prediction rows, prompts, rationales, API checkpoints, SQLite databases,
split manifests, and packet captures are excluded.

| File group | Scope |
|---|---|
| `session_local_*_balanced` | Full held-out folds, balanced test cohorts, five local classifiers |
| `session_local_*_deployment` | Full held-out folds, corpus-prevalence test cohorts, validation-selected thresholds |
| `session_llm_*_pilot_*` | Initial budget-profile GPT-5.4 memory runs |
| `session_llm_*_expanded_*` | Higher-depth GPT-5.4 memory runs used for the current headline comparison |
| `session_llm_author_supplied_window_sweep` | Supplemental complete-session 30 s/1 s-bin GPT-5.4 F1 values; no raw predictions or confusion counts available |
| `phase4e_openai_session_windows` | Preliminary first-5/10/20/50-packet prefix sensitivity check; not capture-disjoint |
| `session_input_requirements` | Frozen 6,000-session deployment-cohort audit, measured full-session encoding structure, and hypothetical first-1 s/5 s/30 s input-volume projections |

Each Phase 7 export records its source artifact name and SHA-256 digest. LLM
exports retain summary, held-out-fold, and malware-family records. Local exports
retain summaries, paired comparisons, and unsupported-status records. The Phase
4E file instead records the recovered source commit and Git blob. The
supplemental window-sweep file explicitly records that its F1-only values were
supplied by the project author and cannot be independently recomputed from this
repository. `session_input_requirements` verifies that current Phase 7 time-bin
variants serialize complete sessions; its prefix records project data volume
only and must not be assigned Phase 7 detector accuracy. Reproducing that
profile requires the excluded local database and frozen manifests. The source
artifacts are intentionally not required to inspect the other published
metrics.

The expanded whole-session headline results are:

| Evaluation | Detector / features | Pooled malicious F1 |
|---|---|---:|
| Balanced | GPT-5.4 / combined | 84.41% |
| Balanced | CART / Mercury-style | 89.08% |
| Deployment | GPT-5.4 / minimal | 90.33% |
| Deployment | Random Forest / minimal | 83.18% |

These are five-capture pooled values, not per-family guarantees. Consult the
fold-level records before citing them; capture heterogeneity is substantial.

Claude Sonnet 4.6 is supported by the code, but no provider-identified Sonnet
session artifact was present in the result archive used for these exports. It is
therefore not assigned metrics in this directory.

The Phase 4E summary is not generated by the exporter because its source belongs
to Git commit `93208e275b62`, not the current `results/` directory. It records the
source Git blob and explicitly preserves the missing model identity and weaker
split protocol. It is genuine first-packet-prefix evidence, unlike Phase 7's
complete-session binning, but it is not pooled with Phase 7. The author-supplied
bin-width sweep is also not generated by the exporter because its prediction-
level artifacts were not provided.

Regenerate the six Phase 7 exports after running experiments:

```powershell
python scripts/export_publishable_results.py
```

Regenerate the input-requirement summary only in the private experiment
workspace containing `traffic.db` and the frozen split manifests:

```powershell
python scripts/profile_session_input_requirements.py
```
